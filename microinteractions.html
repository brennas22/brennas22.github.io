Microinteractions - Case Study

Microinteractions

This case study in microinteractions is a side project that was spurred by a need to learn Principle. The app helps people shop more efficiently for a weeks worth of meals and provides them with recipes, while learning what they do and do not like. While I think the app is an idea potentially worth pursuing, this case study does not include any sort of user validation or testing and is instead focused on creating micro interactions within the design as a means of teaching myself a new technical skill

Along the way I also learned how to use Adobe AfterEffects and photoshop to place gifs inside of mockups, which was... an adventure...

Hover over gifs to play!

I began with the sign-up page, which has the join button sliding into a new position, as well as lines expanding from the left

For search, I animated the hover state of the search button, as well as how it expands into a full search box. I also simulated typing and selecting an item

The dislike page allows users to select items (changing their color to indicate) and then mark their distaste, including allergies. This page features sliders for each item, which are disabled if the user marks an allergy

With the ability selection I played with Principle's "spring" effects, which makes it appear that the selector is bouncing from one item to the next

For the main learning section of the app I went with the familiar Tinder paradigm
